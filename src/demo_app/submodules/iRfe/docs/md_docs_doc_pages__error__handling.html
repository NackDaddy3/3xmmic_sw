Error Handling

The iRFE driver provides error handling in the following ways:

  • Error codes are returned in case of errors for individual function and command calls. These error codes are defined in Ifxrfe_ErrorDefinitions.h
  • Error log messages can be stored by the user with formatting using the error logging functionality provided by errorLogFmt, warningLogFmt and infoLogFmt. The Error logs can be saved directly without additional formatting using errorLog, warningLog and infoLog.

These methods can then be used in the application using the iRFE struct IfxRfe_logCallbacks_t as shown in the below example code.

CTRX firmware errors and their handling

CTRX provides firmware commands to allow error handling of the firmware errors. These are firmware commands: Handle_Error() to handle error and Handle_Reset_Reason() to know reset reason status and clear it if requested. Please refer to the User manual section 4.5 Error status for further details of these firmware errors and commands. These firmware commands can be accessed via iRFE through IfxRfe_handleError and IfxRfe_handleResetReason.

Note
The firmware error handling is not dependent on the iRFE driver and is fixed in firmware. The iRFE driver only provides access to the firmware commands defined for error handling.
