Execute the Configure_And_Execute_Handle() firmware command.

Parameters
handle_idCommand id for new command or handle of the command to be updated
action0 for updating an existing handle, 1 for configuring a new handle
cmdCommand structure already containing the serialized payload
handleHandle of the stored configuration, 0xFF indicates that the configuration has failed
rsp[out] Response returned from CTRX
Return values
IFXRFE_E_SUCCESSon success
IFXRFE_E_NOT_INITIALIZEDif IfxRfe not initialized
IFXRFE_E_INVALID_SIZEif received response length does not match
IFXRFE_E_UNEXPECTED_VALUEif received command id does not match command_id
IFXRFE_E_MISSING_PARAMETERif cmd or rsp is NULL or handle is NULL when the action is "new"
IFXRFE_E_INVALID_PARAMETERif cmd.payloadLen too long
IFXRFE_E_FAILEDif an invalid handle was received
Note
error code from SpiSendReceive otherwise

◆ IfxRfe_fwCall_executeDirectly_read()

error_t IfxRfe_fwCall_executeDirectly_read ( IfxRfe_asyncHandle_t  handle,
IfxRfe_ctrxResponse_t *  rsp 
)

Read back the result of the last Execute_Directly() firmware command.

Note
It is not allowed to execute any commands between the last IfxRfe_fwCall_executeDirectly_send and this call, otherwise the CRI would not match
The function waits for the RFT pin blockingly
Parameters
handleHandle of the last IfxRfe_fwCall_executeDirectly_send call, containing the parameters to check
rsp[out] Response returned from CTRX
Return values
IFXRFE_E_SUCCESSon success
IFXRFE_E_NOT_INITIALIZEDif IfxRfe not initialized
IFXRFE_E_MISSING_PARAMETERif rsp is NULL
IFXRFE_E_INVALID_SIZEif received response too short
IFXRFE_E_UNEXPECTED_VALUEif received command id does not match command_id
IFXRFE_E_CRI_MISMATCHif response cri does not match command cri
IFXRFE_E_OPCODE_MISMATCHif response opcode does not match command opcode
IFXRFE_E_SPI_MSG_ERRORor IFXRFE_E_COMMAND_ERROR if the response indicates one of the errors
Note
error code from SpiRead otherwise
