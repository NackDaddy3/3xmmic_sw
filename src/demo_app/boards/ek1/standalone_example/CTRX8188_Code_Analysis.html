Project: MIMOrad - P2S40_3xCTRX8188 Standalone Example

Platform: NVIDIA Jetson Orin AGX Developer Kit

Radar Board: Infineon P2S40 with 3x CTRX8188 MMIC Radar Chips

Firmware Version: 1.1.0-RamFw_1.0.0-rc3

Analysis Date: July 14, 2026

1. High-Level Overview

This codebase implements a standalone radar demonstration application that initializes and operates a single Infineon CTRX8188 radar chip (CTRX_A) on a board capable of supporting three chips. The primary purpose is to:

  • Initialize the hardware platform - Configure GPIO, SPI, and I2C interfaces on the NVIDIA Jetson Orin
  • Power up the radar system - Enable PMICs (Power Management ICs) with spread spectrum control
  • Configure the radar chip - Load sequencer programs, configure RF parameters, and set up receiver/transmitter chains
  • Execute radar operations - Perform chirp sequences and stream radar data via MIPI CSI-2 interface
  • Provide user interaction - Allow manual control of radar operation cycles through console input
Note: Although the hardware supports 3 radar chips (CTRX_A, CTRX_B, CTRX_C), this example code only initializes and operates CTRX_A in standalone mode. The radar data is streamed through the MIPI CSI-2 interface to the Jetson Orin's camera subsystem, where it can be captured as video device /dev/video0.

2. Step-by-Step Execution Flow

Step 1: Application Initialization

The program starts at main() in main.c:44 and prints a banner message.

printf(" === IfxRfe App - p2s40_3x8188_standalone_example ===\n");

Step 2: PMIC Configuration Objects

Two MAX20434 PMIC structures are created (main.c:49-50):

  • pmic_ab (I2C address 0x3E) - Powers CTRX_A and CTRX_B
  • pmic_c (I2C address 0x3F) - Powers CTRX_C

Step 3: GPIO Configuration (main.c:55-78)

Configure all necessary GPIO pins using PlatformGpio_configure():

  • General GPIOs:
    • GPIO_ID_EN_PMIC - PMIC enable (output, initially LOW)
    • GPIO_ID_CTRX_RESET_N - Radar chip reset (output, initially LOW)
    • GPIO_ID_CTRX_OK - Radar chip status indicator (input)
    • GPIO_ID_PGOOD_PMIC_LV/HV - PMIC power good signals (inputs)
  • CTRX_A specific GPIOs:
    • GPIO_ID_RFT_A - RFT signal (input)
    • GPIO_ID_SPI_SS_A - SPI chip select (output, HIGH)
    • GPIO_ID_DMUX_1_A - Data multiplexer 1 (input)
    • GPIO_ID_DMUX_2_A - Data multiplexer 2 (input)
  • CTRX_B and CTRX_C: Similar GPIO configurations (not used in standalone example)

Step 4: I2C Initialization (main.c:81)

Initialize I2C bus ID 2 (I2C3 in hardware) for PMIC communication:

PlatformI2c_init(I2C_ID);

Step 5: SPI Configuration (main.c:85-93)

Configure SPI for communication with CTRX_A:

  • Device ID: 0
  • SPI ID: CTRX_A_SPI_ID (0x10)
  • CS GPIO: GPIO_ID_SPI_SS_A
PlatformSpi_init(); PlatformSpi_setDevIdSpiIdMapping(devIds, spiIds, sizeof(devIds)); PlatformSpi_setCsGpioIds(spiIds, spiCsGpioIds, sizeof(spiIds));

Step 6: IfxRfe Driver Setup (main.c:98-140)

Initialize the Infineon RFE (Radio Frequency Engine) driver with platform-specific function pointers:

  • SPI Functions: spiConfigure, spiTransfer, spiWrite
  • GPIO Functions: gpioGet, gpioSet, gpioConfigure
  • Time Functions: getDeadLine, now, waitTime
  • Log Callbacks: errorLog, warningLog, infoLog
IfxRfe_init(CTRX_DEV_COUNT, gpios, spiFncs, gpioFncs, timeFncs, logInterface);

Step 7: Reset and Power Sequence (main.c:143-169)

  1. Assert CTRX reset (LOW) - main.c:144
  2. Enable PMICs - main.c:148
  3. Configure spread spectrum (±3% pseudo-random) on both PMICs - main.c:153-155
  4. Reset PMIC outputs (workaround for Jetson leakage issue) - main.c:160-164
  5. De-assert CTRX reset (HIGH) to start radar chips - main.c:168

Step 8: CTRX_A Initialization (main.c:176-195)

Initialize demo configuration parameters and prepare CTRX initialization structure:

  • Load configuration from IrfeDemoConfigInit() (defined in config.c)
  • Set IRAM descriptors for standalone mode, CRC32 disabled, continuous CSI-2 clock
  • Configure SPI clock speed: 3.125 MHz
  • Execute IfxRfe_ctrxInit() to initialize the radar chip
static uint32_t intitializeIramDesc[] = {0x10000028, 0x1180000A, 0x1000000E}; static uint32_t intitializeValue[] = {1, 2, 1}; // Standalone / CRC32 disabled / continuous CSI-2 clk

Step 9: Sequencer Program Loading (main.c:199-202)

Load the radar sequencer program (512 ramps, 512 samples per ramp, 8 RX channels):

IfxRfe_loadSequencerData(0, sequencer_program_data, sizeof(sequencer_program_data) / sizeof(uint32_t));

This defines the chirp sequence: 80 GHz center frequency, 600 MHz bandwidth, 10 µs chirp time, 512 ramps, TX1 active

Step 10: Radar Configuration (main.c:205-226)

Configure radar parameters sequentially:

  1. DMUX Configuration: Configure data multiplexer for ramp scenario completion detection
  2. Clear Results: Clear previous measurement results
  3. Ramp Scenario: Configure ramp scenario with sequencer start offset
  4. TX Power: Set transmit power levels for all TX channels
  5. RX Configuration: Configure receiver frontend (0dB gain, 12-bit data, 1200 Mbps data rate)
  6. RF Frequency: Configure RF frequency parameters (NCW, NMOD, ramp bandwidth)

Step 11: Temperature Reading (main.c:229-232)

Read and display chip temperatures from two internal sensors:

IfxRfe_getTemperature(&tempRes); printf(" Temperature Sensor 1: %.1f °C\n", ((float)tempRes.temp1) / 8.0f); printf(" Temperature Sensor 2: %.1f °C\n", ((float)tempRes.temp2) / 8.0f);

Step 12: Transition to Operation Mode (main.c:235-239)

Move the radar chip from configuration state to operational state:

IfxRfe_gotoOperation(); // Verify state is 0x10000000 (operational)

Step 13: TX Calibration (main.c:242-244)

Execute transmitter calibration subfunctions (all TX calibration functions enabled with mask 0x000F):

_configParams.calibration.calib_sub_func_id = 0x000F; IfxRfe_executeCalibration(_configParams.calibration, NULL);

Step 14: DMUX Pre-Check (main.c:247)

Wait for DMUX_1_A to be LOW before starting ramp scenarios:

PlatformGpio_gpioWait(GPIO_ID_DMUX_1_A, false, 10);

Step 15: Radar Operation Loop (main.c:250-268)

Enter an interactive loop (up to 10,000 iterations) where the user controls execution:

  1. User Prompt: Ask user to continue ('c') or exit ('e')
  2. Start Ramp Scenario: IfxRfe_startRampScenario() - Begins chirp transmission and data capture
  3. Wait for Completion: PlatformGpio_gpioWait(GPIO_ID_DMUX_1_A, true, 15) - Wait for DMUX signal indicating completion
  4. Finish Ramp: IfxRfe_finishRampScenario(NULL) - Complete the ramp scenario

Data Capture: While the radar executes chirps, data is streamed via MIPI CSI-2 to the Jetson Orin. Use the provided shell script captureA_8x512x512.sh to capture data using v4l2-ctl from /dev/video0.

Step 16: Cleanup and Exit (main.c:271-274)

Call cleanup function to properly deinitialize all hardware:

cleanup_platform(); // Resets CTRX, deinitializes I2C, SPI, GPIO printf(" === SUCCESS ===\n");

3. Key Components and Their Roles

3.1 main.c

Role: Application entry point and orchestrator

  • Initializes all hardware interfaces (GPIO, SPI, I2C)
  • Manages power sequencing for radar chips
  • Configures and operates the radar in a loop
  • Provides cleanup mechanisms

3.2 config.c / config.h

Role: Radar parameter configuration

Function: IrfeDemoConfigInit() - Initializes radar configuration structure with:

  • RF Frequency Parameters:
    • NMOD: 62432208 (frequency modulation parameter)
    • NCW: 283073584 (center wave number)
    • Ramp BW: 8738128 (approximately 2.5 GHz modulation bandwidth)
  • TX Power Configuration: Power levels for 4 TX channels (all set to 0)
  • RX Configuration:
    • Gain: 0dB
    • Data width: 12 bits
    • Data rate: 1200 Mbps
    • Start mode: Immediate
  • Calibration Settings: TX channel power calibration parameters
  • DMUX Configuration: Data multiplexer for scenario completion signaling

3.3 HwDefinitions.h

Role: Hardware abstraction and GPIO mapping

Defines the mapping between logical GPIO IDs and physical Jetson Orin pins:

  • GPIO Port Offsets: Defines offsets for ports A-AG, AA-GG
  • Chip-Specific GPIO IDs: Maps signals for each CTRX chip (A, B, C)
  • SPI IDs: CTRX_A (0x10), CTRX_B (0x11), CTRX_C (0x12)
  • I2C Bus: I2C ID 2 (physical I2C3)

3.4 Platform Layer (platform-jetson submodule)

Role: Hardware abstraction layer for Jetson Orin

PlatformSpi.c / PlatformSpi.h

  • Implements SPI communication using Linux spidev driver
  • Manages manual chip-select control via GPIO (required for keeping CS asserted during multi-frame transactions)
  • Supports device ID to SPI ID mapping

PlatformGpio.c / PlatformGpio.h

  • GPIO control using libgpiod v1.6
  • Manages two GPIO chips: gpiochip0 (GPIOs 348-511) and gpiochip1 (GPIOs 316-347)
  • Provides GPIO waiting/polling mechanisms

PlatformI2c.c / PlatformI2c.h

  • I2C communication via Linux SMBus interface
  • Used for PMIC control

3.5 IfxRfe Driver (iRfe submodule)

Role: Infineon Radio Frequency Engine middleware

Provides high-level API for controlling CTRX8188 radar chips:

  • IfxRfe_init() - Initialize driver with platform callbacks
  • IfxRfe_ctrxInit() - Initialize radar chip
  • IfxRfe_loadSequencerData() - Load chirp sequencer program
  • IfxRfe_configureRampScenario() - Configure ramp parameters
  • IfxRfe_configureTxPower() - Set TX power levels
  • IfxRfe_configureRx() - Configure receiver frontend
  • IfxRfe_configureRfFrequency() - Set RF frequency
  • IfxRfe_gotoOperation() - Transition to operational state
  • IfxRfe_executeCalibration() - Run calibration routines
  • IfxRfe_startRampScenario() - Begin chirp execution
  • IfxRfe_finishRampScenario() - Complete chirp execution

3.6 PMIC Driver (Max20434Pec.c)

Role: Power management IC control

Controls MAX20434 PMICs via I2C:

  • Max20434_enableSpreadSpectrum() - Enable ±3% spread spectrum for EMI reduction
  • Max20434_setManualOutputEnable() - Control PMIC output enable

3.7 3xCtrx8188Wrapper.c

Role: SPI override wrappers

Provides board-specific SPI transfer and write function overrides:

  • PlatformSpi_transfer_override()
  • PlatformSpi_write_override()

3.8 captureA_8x512x512.sh

Role: Data capture script

Captures radar data from MIPI CSI-2 interface using v4l2-ctl:

  • Device: /dev/video0
  • Format: RG12 (RAW12 pixel format), 4096x512 pixels
  • Output: Binary raw file and hexdump

7. Key Technical Details

7.1 Radar Configuration

  • Center Frequency: ~80 GHz (E-band automotive radar)
  • Modulation Bandwidth: ~2.5 GHz
  • Chirp Time: 10 µs per chirp
  • Ramps per Frame: 512
  • Samples per Ramp: 512
  • Active RX Channels: 8
  • Active TX Channels: 1 (TX1)
  • Data Width: 12 bits per sample

7.2 Data Interface

  • Protocol: MIPI CSI-2 v1.3
  • Number of Lanes: 4
  • Data Rate: 1200 Mbps per lane
  • Clock Mode: Continuous
  • Pixel Format: RAW12 (12-bit samples with 4-bit padding to 16-bit)
  • Frame Size: 4096 × 512 pixels (representing 8 RX channels × 512 samples × 512 ramps)
  • Linux Device: /dev/video0 (captured via V4L2 subsystem)

7.3 Timing and Synchronization

  • DMUX_1_A Signal: Used to detect ramp scenario completion
    • LOW = Idle/Ready
    • HIGH = Ramp scenario executing
  • Power-Up Delays:
    • 10 ms after reset assertion
    • 10 ms after PMIC enable
    • 100 ms during PMIC output reset workaround
    • 10 ms after reset de-assertion

7.4 Error Handling

The code uses two macros for error checking:

  • EXIT_ON_PLATFORM_ERROR() - Checks platform layer errors and calls cleanup on failure
  • EXIT_ON_IFXRFE_ERROR() - Checks IfxRfe driver errors and calls cleanup on failure

Both macros ensure proper resource deallocation before exit.

8. Data Capture Procedure

Step 1: Start the Radar Application

$ ./p2s40_3x8188_standalone_example

Step 2: Wait for "Enter 'c' to continue" Prompt

The radar is now initialized and in operational mode with MIPI CSI-2 clock active.

Step 3: In a Separate Terminal, Run the Capture Script

$ ./captureA_8x512x512.sh

This script executes:

v4l2-ctl -d /dev/video0 \ --set-fmt-video=width=4096,height=512,pixelformat=RG12 \ --set-ctrl bypass_mode=0 \ --stream-mmap \ --stream-count=1 \ --stream-to=ctrx1_bin.raw

Step 4: In the Radar Application Terminal, Press 'c' and Enter

This triggers the radar to execute a ramp scenario and stream data.

Step 5: Captured Files

  • ctrx1_bin.raw - Raw binary radar data (4096 × 512 × 2 bytes = 4,194,304 bytes)
  • ctrx1_hex_packed.txt - Hexdump of the captured data
Important: The CSI-2 receiver on the Jetson will only activate if the CTRX is in operational mode and transmitting the CSI-2 clock. Therefore, the capture script must be started after the radar application has reached the operational state.

9. Summary

This standalone example demonstrates a complete radar system integration between an Infineon CTRX8188 MMIC radar chip and the NVIDIA Jetson Orin platform. The code provides:

  • Hardware abstraction through the platform-jetson layer
  • Radar configuration via the IfxRfe middleware
  • Power management through I2C-controlled PMICs
  • High-speed data streaming via MIPI CSI-2 to the Jetson's camera subsystem
  • User-controlled operation with interactive radar execution loops

The system leverages the Jetson Orin's camera connector (J509) which provides all necessary interfaces: SPI for control, I2C for power management, GPIO for status monitoring, and MIPI CSI-2 for high-bandwidth radar data streaming. This allows the radar data to be captured as standard video frames using Linux V4L2 tools, simplifying data acquisition and processing.

The architecture is designed to be extensible to support all three CTRX chips (A, B, C) for MIMO radar applications, though this standalone example focuses on a single chip to demonstrate the fundamental operation.
